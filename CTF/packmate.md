# Настройка

В .env настраиваем IP и интерфейс тот, который принимает трафик (на котором развернуто приложение).

Там же настраиваем имя пользователя и пароль для подключения.

# Пересобрать

```
docker compose up -d --build
```

# Завести приложение

Нажимаем плюсик. Указываем параметры:

- Имя
- Порт (если сервис использует несколько портов, нужно создать по сервису на каждый порт)
- Is an HTTP service: если веб-приложение
- Urldecode: автоматически проводить urldecode пакетов. Стоит включать по умолчанию в http сервисах.
- Merge adjacent packets: автоматически склеивать соседние пакеты в одном направлении. Стоит включать по умолчанию в небинарных сервисах.
- Inflate WebSockets: автоматически разархивировать сжатые websocket-пакеты.
- Decrypt TLS: автоматически расшифровывать TLS-трафик (HTTPS). Работает только с типами шифрования TLS_RSA_WITH_AES_*, и при наличии приватного ключа, который использовался в сертификате сервера (как Wireshark).

# Доп фичи

Слева сверху есть 3 кнопки. Две из них достаточно важные:

- старт/стоп - можно остановить сбор пакетов, чтобы лучше проанализировать. Потом запускаем обратно, чтобы понимать, что попатчили.
- звездочка - можно помечать интересные пакеты, а потом открывать их тут. Удобно, когда понял, в каких местах уязвы, сохранил - по очереди смотришь их.
